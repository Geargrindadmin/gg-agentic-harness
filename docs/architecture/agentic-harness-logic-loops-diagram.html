End-to-End System Map (Processes + Logic Loops)

Internal Loop (In-Run)

Purpose: stabilize one run using bounded retries and deterministic evidence.

A. Gate failure capture
Normalize failure signature and append to run artifact.
B. Retry budget
Retry with backoff (1s -> 2s -> 4s), hard cap at policy limit.
C. Recurring threshold
Same signature 2+ times triggers recurring-failure evidence.
D. Recovery action
Create discovered-from hardening follow-up; avoid silent loops.

External Loop (Cross-Run)

Purpose: upgrade the system itself, not just the current ticket.

A. Aggregate run artifacts
Scan recent runs for repeated failure signatures.
B. Detect patterns
Threshold-based detection of systematic issues.
C. Propose hardening
Emit proposal docs for rule/skill/workflow/persona updates.
D. Govern + merge
Board decision + ADR + harness contract integration.
