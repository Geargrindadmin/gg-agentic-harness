New Headless Systems

Executable `create` surface Takes prompt, PRD path, or canonical spec and emits a deterministic application bundle with manifest and README.
Shared `product-builder.ts` First real bundle generator in the repo. Produces supported starter bundles for the mandatory V1 lanes.
Shared `product-spec.ts` Canonical spec normalization now lives in core instead of inside the CLI file.
Shared `preflight.ts` Runtime activation, parity, and project-context checks now have one reusable contract.
Executable `agentic-status` Reports live activation, parity, recent runs, control-plane state, and overall repo readiness.
Portable bundle proof Live bundle exists at .agent/product-bundles/live-marketing-bundle/.

Why This Matters

Headless by default The harness can now emit a concrete product package without requiring an IDE panel or conversational babysitting.
Deterministic ownership preserved `go`, `minion`, and `agentic-status` still fail fast when the runtime is not in a trusted runnable state.
Downstream install stays separate The harness is proved in its own repo first, then bundles can be installed into `GGV3`.
Supports enterprise framing Lanes, packs, gates, constraints, and integration requirements are all visible in the bundle manifest rather than hidden in prose.
