# Front-End Checklist Skill

This skill provides a comprehensive checklist for frontend development quality assurance. Use it before any frontend launch, during UI audits, or when building new pages.

## Reference

See [checklist-reference.md](./checklist-reference.md) for the full checklist covering:
- **Head**: Meta tags, social meta, HTML tags
- **HTML**: Best practices, testing
- **Webfonts**: Loading strategies, formats
- **CSS**: Methodology, performance, testing
- **Images**: Optimization, formats, best practices
- **JavaScript**: Best practices, testing
- **Security**: Headers, HTTPS, CSP, scanning
- **Performance**: Core Web Vitals, lazy loading, caching
- **Accessibility**: WCAG, semantics, forms, testing
- **SEO**: Meta, structured data, sitemap, robots

## Usage

When triggering this skill:
1. Determine which sections are relevant to the current task
2. Run through each checklist item in the relevant sections
3. Report findings as PASS/FAIL/N-A
4. Prioritize Critical items first, then High, then Medium
